## Directory Purposes

**config/:**
- Purpose: Package configuration files published to consuming application
- Contains: Configuration arrays with environment overrides
- Key files: `config/model-explorer.php`

**database/:**
- Purpose: Database utilities for testing
- Contains: Eloquent factories and migrations
- Key files: None currently (directory structure exists for future use)

**docs/:**
- Purpose: Project documentation
- Contains: Architecture Decision Records and other docs
- Key files: ADR documents in `docs/adr/`

**public/:**
- Purpose: Compiled frontend assets served to browsers
- Contains: Built JavaScript and CSS from Vite
- Key files: `app.js`, `app.css` (generated by Vite build)

**resources/css/:**
- Purpose: Source stylesheets
- Contains: CSS files for styling the SPA
- Key files: `app.css`

**resources/js/:**
- Purpose: Source JavaScript and Vue components
- Contains: Vue 3 components and initialization scripts
- Key files: `app.js` (entry point), `App.vue` (root component)

**resources/views/:**
- Purpose: Blade templates for server-side rendering
- Contains: HTML shell for SPA
- Key files: `app.blade.php` (page shell)

**routes/:**
- Purpose: Route definitions
- Contains: Web route handlers and middleware application
- Key files: `web.php`

**src/Facades/:**
- Purpose: Facade pattern implementations for service access
- Contains: Facades wrapping core services
- Key files: `ModelExplorer.php`

**src/Http/Controllers/:**
- Purpose: HTTP request handlers
- Contains: Controller classes for routes
- Key files: `ModelExplorerController.php`, `AssetController.php`

**src/Http/Middleware/:**
- Purpose: HTTP request/response middleware
- Contains: Middleware for cross-cutting concerns
- Key files: `Authorize.php`

**src/:**
- Purpose: Core package logic
- Contains: Service classes and service provider
- Key files: `LaravelModelExplorerServiceProvider.php`, `LaravelModelExplorer.php`

**tests/Feature/:**
- Purpose: Feature-level integration tests

- Key files: `RouteAuthorizationTest.php`

**tests/:**
- Purpose: Test suite
- Contains: Test classes and configuration

## Naming Conventions

**Files:**
- PHP classes: `PascalCase.php` (e.g., `ModelExplorerController.php`)
- Blade views: `kebab-case.blade.php` (e.g., `app.blade.php`)
- Vue components: `PascalCase.vue` (e.g., `App.vue`)
- Configuration files: `kebab-case.php` (e.g., `model-explorer.php`)
- JavaScript modules: `camelCase.js` (e.g., `app.js`)

**Directories:**
- PHP namespaces: PascalCase (e.g., `Http`, `Controllers`, `Middleware`)
- Feature directories: kebab-case (e.g., `resources`, `public`)
- Test directories: Follow structure of src (e.g., `tests/Feature/`)

**PHP Classes:**
- Controllers: `{Name}Controller` suffix (e.g., `ModelExplorerController`)
- Middleware: Plain class name (e.g., `Authorize`)
- Facades: Plain class name (e.g., `ModelExplorer`)
- ServiceProviders: `{Name}ServiceProvider` suffix (e.g., `LaravelModelExplorerServiceProvider`)

**Routes:**
- Route names: kebab-case with dot notation (e.g., `model-explorer.index`, `model-explorer.assets.serve`)
- Route paths: kebab-case prefixes (e.g., `_model-explorer`, `_model-explorer/assets`)

## Where to Add New Code

**New Feature:**
- Primary code: `src/` (in appropriate subdirectory like `Services/`, `Actions/`)
- Tests: `tests/Feature/` for integration tests or `tests/Unit/` for unit tests
- Routes: Add to `routes/web.php` if HTTP endpoint needed

**New Frontend Component:**
- Implementation: `resources/js/` (create subdirectories for organization, e.g., `components/`)
- Entry point update: Register in `App.vue` or use lazy loading

**New Frontend Page/View:**
- Implementation: `resources/js/Pages/` (Vue 3 component)
- Routes: Manage via client-side router (not Laravel routes)

**Utilities/Helpers:**
- Shared PHP helpers: `src/Support/` or similar
- Shared Vue utilities: `resources/js/utils/` or `resources/js/composables/`

**Configuration:**
- Package config: `config/model-explorer.php` (published to consuming app)
- Runtime config: Use `config()` helper throughout code

**Tests:**
- Feature tests: `tests/Feature/{Feature}Test.php`
- Unit tests: `tests/Unit/{Unit}Test.php`
- Base class: Extend `Tests\TestCase`

## Package Integration Points

**Service Provider:**
- Registered in Laravel's service container via `src/LaravelModelExplorerServiceProvider.php`
- Publishes: config files, routes, views
- Registers: Gate for authorization

**Routes:**
- Registered via service provider's `hasRoutes('web')` call
- Prefixed by `config('model-explorer.path')` (default: `_model-explorer`)
- Two route groups: assets (public) and SPA (protected)

**Views:**
- Registered via service provider's `hasViews()` call
- Namespace: `model-explorer`
- Access: `view('model-explorer::app')`

**Configuration:**
- Published to `config/model-explorer.php` in consuming app
- Customizable options: enabled flag, path, middleware, model_paths
